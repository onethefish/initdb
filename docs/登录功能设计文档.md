# InitDB-AI 登录功能设计文档

## 1. 概述

### 1.1 文档信息
| 项目名称 | InitDB-AI 智能数据库助手 |
|---------|-------------------------|
| 文档版本 | v1.0 |
| 创建日期 | 2026-04-26 |
| 作者     | Claude |

### 1.2 需求背景
当前系统缺少用户管理功能，所有用户均可无限制访问系统资源。为实现多用户隔离、数据安全及权限控制，需接入登录认证功能。

### 1.3 功能范围
- 账号密码登录
- 短信/邮箱验证码登录
- OAuth 2.1 第三方登录（支持微信、GitHub、Google 等）
- Token 管理与刷新机制

## 2. 技术选型

### 2.1 认证框架
| 组件 | 选型 | 说明 |
|------|------|------|
| 认证框架 | Spring Security 6.x | Spring 官方安全框架，与 Spring Boot 3.x 完美集成 |
| Token 方案 | JWT (JSON Web Token) | 无状态认证，适合前后端分离架构 |
| OAuth 实现 | Spring Authorization Server | Spring 官方 OAuth 2.1 授权服务器实现 |
| 验证码 | Redis + 自定义实现 | 验证码存储与过期控制 |
| 密码加密 | BCrypt | 业界标准密码哈希算法 |

## 4. 数据库设计

### 4.1 用户表 (sys_user)
```sql
CREATE TABLE sys_user (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    username        VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password        VARCHAR(255) COMMENT '密码(BCrypt加密)',
    nickname        VARCHAR(100) COMMENT '昵称',
    email           VARCHAR(100) UNIQUE COMMENT '邮箱',
    phone          VARCHAR(20) UNIQUE COMMENT '手机号',
    avatar         VARCHAR(255) COMMENT '头像URL',
    status         TINYINT DEFAULT 1 COMMENT '状态: 0-禁用, 1-启用',
    create_time    DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time    DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    last_login_time DATETIME COMMENT '最后登录时间',
    INDEX idx_username (username),
    INDEX idx_phone (phone),
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 4.2 第三方账号绑定表 (sys_user_oauth)
```sql
CREATE TABLE sys_user_oauth (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    user_id         BIGINT NOT NULL COMMENT '用户ID',
    provider        VARCHAR(20) NOT NULL COMMENT 'OAuth提供商: WECHAT/GITHUB/GOOGLE',
    provider_user_id VARCHAR(100) NOT NULL COMMENT '第三方用户ID',
    union_id        VARCHAR(100) COMMENT '统一用户标识(微信)',
    open_id         VARCHAR(100) COMMENT '开放平台标识',
    access_token    VARCHAR(500) COMMENT '第三方访问令牌',
    refresh_token   VARCHAR(500) COMMENT '第三方刷新令牌',
    expires_at      DATETIME COMMENT '令牌过期时间',
    create_time     DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '绑定时间',
    UNIQUE KEY uk_provider_user (provider, provider_user_id),
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='第三方账号绑定表';
```

### 4.3 验证码记录表 (sys_captcha)
```sql
CREATE TABLE sys_captcha (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    target          VARCHAR(100) NOT NULL COMMENT '目标(手机号/邮箱)',
    captcha        VARCHAR(10) NOT NULL COMMENT '验证码',
    type           VARCHAR(20) NOT NULL COMMENT '类型: LOGIN/REGISTER/RESET',
    status         TINYINT DEFAULT 0 COMMENT '状态: 0-未使用, 1-已使用',
    expire_time    DATETIME NOT NULL COMMENT '过期时间',
    create_time    DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    INDEX idx_target_type (target, type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='验证码记录表';
```

### 4.4 Token 记录表 (sys_token)
```sql
CREATE TABLE sys_token (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    user_id         BIGINT NOT NULL COMMENT '用户ID',
    access_token    VARCHAR(500) NOT NULL COMMENT '访问令牌',
    refresh_token   VARCHAR(500) COMMENT '刷新令牌',
    token_type      VARCHAR(20) DEFAULT 'Bearer' COMMENT '令牌类型',
    expires_in      BIGINT COMMENT '有效期(秒)',
    scope          VARCHAR(255) COMMENT '权限范围',
    create_time    DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    expires_at     DATETIME COMMENT '过期时间',
    INDEX idx_access_token (access_token(100)),
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='Token记录表';
```

## 5. API 接口设计

### 5.1 登录相关接口

#### 5.1.1 账号密码登录
```
POST /api/v1/auth/login
Content-Type: application/json

Request:
{
    "username": "admin",
    "password": "password123",
    "captchaKey": "uuid-xxx",      // 可选，开启图形验证码时需要
    "captchaCode": "1234"          // 可选，图形验证码
}

Response:
{
    "code": 200,
    "message": "success",
    "data": {
        "accessToken": "eyJhbGciOiJIUzI1NiIs...",
        "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
        "tokenType": "Bearer",
        "expiresIn": 7200,
        "userId": 1,
        "username": "admin",
        "nickname": "管理员"
    }
}
```

#### 5.1.2 发送验证码
```
POST /api/v1/auth/captcha/send
Content-Type: application/json

Request:
{
    "target": "13800138000",    // 手机号或邮箱
    "type": "LOGIN",            // LOGIN/REGISTER/RESET
    "captchaKey": "uuid-xxx",   // 图形验证码Key（防止刷验证码）
    "captchaCode": "1234"       // 图形验证码
}

Response:
{
    "code": 200,
    "message": "验证码发送成功"
}
```

#### 5.1.3 验证码登录
```
POST /api/v1/auth/login/captcha
Content-Type: application/json

Request:
{
    "target": "13800138000",    // 手机号或邮箱
    "captcha": "123456",
    "type": "PHONE"             // PHONE/EMAIL
}

Response:
{
    "code": 200,
    "message": "success",
    "data": {
        "accessToken": "eyJhbGciOiJIUzI1NiIs...",
        "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
        "tokenType": "Bearer",
        "expiresIn": 7200,
        "userId": 1,
        "username": "admin"
    }
}
```

#### 5.1.4 Token 刷新
```
POST /api/v1/auth/token/refresh
Content-Type: application/json

Request:
{
    "refreshToken": "eyJhbGciOiJIUzI1NiIs..."
}

Response:
{
    "code": 200,
    "message": "success",
    "data": {
        "accessToken": "eyJhbGciOiJIUzI1NiIs...",
        "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
        "tokenType": "Bearer",
        "expiresIn": 7200
    }
}
```

#### 5.1.5 退出登录
```
POST /api/v1/auth/logout
Authorization: Bearer {accessToken}

Response:
{
    "code": 200,
    "message": "退出成功"
}
```

### 5.2 OAuth 2.1 授权接口

#### 5.2.1 获取授权码
```
GET /oauth2/authorize?response_type=code&client_id=client&scope=read&redirect_uri=https://example.com/callback&state=xyz

Response: 重定向到 redirect_uri?code=xxx&state=xyz
```

#### 5.2.2 获取 Token
```
POST /oauth2/token
Content-Type: application/x-www-form-urlencoded

Request:
grant_type=authorization_code&code=xxx&redirect_uri=https://example.com/callback&client_id=client&client_secret=secret

Response:
{
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "Bearer",
    "expires_in": 7200,
    "scope": "read"
}
```

#### 5.2.3 第三方登录（微信/GitHub/Google）
```
GET /api/v1/oauth2/authorize/{provider}
例如: /api/v1/oauth2/authorize/github

Response: 重定向到第三方授权页面
```

#### 5.2.4 第三方登录回调
```
GET /api/v1/oauth2/callback/{provider}?code=xxx&state=xxx

Response: 重定向到前端页面，携带 Token
```

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/v1/auth/**", "/oauth2/**").permitAll()
                .requestMatchers("/api/v1/**").authenticated()
            )
            .addFilterBefore(jwtAuthenticationFilter(),
                UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

## 8. 安全设计

### 8.1 密码安全
- 使用 BCrypt 算法加密存储，cost factor 设置为 12
- 密码强度要求：至少8位，包含大小写字母、数字
- 登录失败限制：5次失败后锁定账户15分钟

### 8.2 Token 安全
| 配置项 | 值 | 说明 |
|-------|-----|------|
| Access Token 有效期 | 2小时 | 短期有效，减少泄露风险 |
| Refresh Token 有效期 | 7天 | 用于刷新 Access Token |
| Token 签名算法 | RS256 | 非对称加密，更安全 |
| Token 存储 | Redis | 支持主动失效 |

### 8.3 验证码安全
- 验证码有效期：5分钟
- 发送频率限制：同一手机号/邮箱 60秒内只能发送1次
- 每日发送上限：同一手机号/邮箱 每日最多10次
- 验证码长度：6位数字

### 8.4 OAuth 2.1 安全增强
- 强制使用 PKCE (Proof Key for Code Exchange)
- 禁用隐式授权 (Implicit Grant)
- 要求使用 State 参数防止 CSRF
- Redirect URI 严格匹配

## 10. 前端集成说明

### 10.1 Token 存储
```javascript
// 存储 Token
localStorage.setItem('accessToken', response.accessToken);
localStorage.setItem('refreshToken', response.refreshToken);

// 请求拦截器添加 Token
axios.interceptors.request.use(config => {
    const token = localStorage.getItem('accessToken');
    if (token) {
        config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
});
```

### 10.2 Token 刷新机制
```javascript
// 响应拦截器处理 Token 过期
axios.interceptors.response.use(
    response => response,
    async error => {
        if (error.response?.status === 401) {
            const refreshToken = localStorage.getItem('refreshToken');
            if (refreshToken) {
                try {
                    const response = await axios.post('/api/v1/auth/token/refresh', {
                        refreshToken
                    });
                    localStorage.setItem('accessToken', response.data.accessToken);
                    error.config.headers.Authorization = `Bearer ${response.data.accessToken}`;
                    return axios.request(error.config);
                } catch (e) {
                    // 刷新失败，跳转登录页
                    window.location.href = '/login';
                }
            }
        }
        return Promise.reject(error);
    }
);
```
